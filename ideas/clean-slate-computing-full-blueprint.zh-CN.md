# 全新信息计算体系的完整规划蓝图

[English](clean-slate-computing-full-blueprint.md)` | 中文`

> 收录于 2026 年 6 月。讨论性质,前提是先忽略硬件设备和生产制造的问题。
>
> 这是在 [clean-slate-computing-stack](clean-slate-computing-stack.zh-CN.md) 基础上的深化版本。两点关键修正:
> 1. **总纲改为「性能优先 + 清除历史包袱造成的附带复杂度」**,安全是干净设计的副产品,只在真冲突时明说取舍;
> 2. **芯片是全新设计、与微架构协同,只借鉴 RISC-V 的方法论,不继承它的预设。**

## 零、整个体系的「宪法」:五条总原则

在分层之前,先定五条贯穿所有层的元原则。

1. **协同设计,而非事后分层。** 分层仍然存在(为了人能理解),但**层与层之间的接口是一起设计出来的**,不是各做各的再逆向拼接。今天大量附带复杂度来自「接口是历史偶然」。
2. **性能是多轴的,按「复利」排序。** 优先级:**能效(每焦耳有用计算)> 可扩展性(并行/规模)> 延迟 > 峰值吞吐**。前两者会随规模复利,最该优先;主频式的峰值性能放最后。
3. **让「最快的路」就是「最对的路」。** 设计成「性能最高的写法恰好也是安全/正确的写法」,这样就**不存在永久的安全税**——安全不是额外开销,而是干净结构的自然结果。
4. **低层放机制,高层放策略。** 底层只固化最小、最通用的机制;一切会演化的「策略」留在可替换的软层。
5. **层间契约必须显式、可度量、可验证。** 每个接口都有形式化契约,既能跨层验证正确性,也能跨层做优化。

## 一、指令集与芯片:全新设计,与微架构协同

**从 RISC-V 借的方法论(只借这些):** 开放免授权、模块化、从第一天就有形式化规范和「黄金参考模型」、干净规整的编码。

**坚决不继承的 RISC-V 预设:**

- **目标定位反过来。** 新 ISA **从一开始就瞄准现代宽发射乱序 + SMT + 异构**,指令集和微架构一起设计,而不是定死指令集再让硬件迁就。
- **用「档位(Profile)」消灭碎片化。** 不走「最小基础集 + 一堆可选扩展」那条会碎片化的路。改为定义**少数几个完整档位**,每个档位是一整套保证齐全的能力集,不存在拼出来的残缺组合。
- **编码为现代解码器和宏融合而设计。** 编码宽度、对齐、可并行解码性,都和解码器一起定。
- **只有一个强内存一致性模型。** 不给「弱/强可选」。选一个**既高性能又可推理**的模型(release-acquire 体系),并形式化规范死。
- **数据并行/向量/张量是一等公民,不是迟来的扩展。** 向量长度无关(VLA)+ 张量原语从 day one 就和标量统一设计。
- **能力寻址原生进硬件(CHERI 路线),但理由是性能。** 指针自带边界/标签,越界在硬件上不可能发生。**关键卖点不是「更安全」,而是它替软件省掉了一大类运行时检查**——性能 + 正确性双赢;代价(指针变宽、标签位、少量硅面积)明说。
- **异构统一寻址。** CPU/GPU/NPU/DPU **共享同一个相干、字节寻址的虚拟地址空间**,加速器平权、直接访问同一份内存,取消「跨 PCIe 来回拷贝」的税。

## 二、内存与互连:把「零件动物园」收敛成一张协同的网

- **统一相干内存织物。** 所有计算单元看到同一个相干、字节寻址的内存空间;**近存/存内计算原语**直接暴露给软件。
- **存储层级用「一个地址空间 + 放置提示」表达。** 高带宽层、主存层、持久层都在同一地址空间里,靠显式 placement hint 调度。
- **互连只有一张网。** 把今天 PCIe / 以太 / NVLink / CXL 这一堆各自为政的标准,**合并成一张协同设计的、自带相干性和身份的交换织物**,从片内一路连续到机架、到数据中心。**内存互连和网络是一条连续谱。**

## 三、操作系统:形式化验证的微内核,但为性能而调

- **最小可信基 + 形式化验证内核(seL4 级)**,但**生死线是 IPC 性能**:把**零拷贝的「能力传递式 IPC」**做成内核的中心原语。
- **全栈能力化,且与硬件能力同构。** OS 的能力直接映射到硬件能力,不是软件模拟出来的。
- **默认异步、消息传递、批量提交。** 取消「同步系统调用陷入」作为热路径默认;**io_uring 式的批量异步提交/完成**成为原生 ABI。
- **声明式、可复现的系统状态(Nix 思想)是原生配置模型**,整机状态是可回滚、可复现的描述。
- **调度器懂异构、懂能耗。** 在大核/能效核/加速器之间调度时,**把能耗当成一等优化目标**。
- 放弃「一切皆文件」的过度泛化,改用**带能力的类型化对象**;保留一个小而统一的命名机制。

## 四、网络:身份与加密是协议原语

- **每个端点有密码学身份,每个包天然可认证,端到端加密是底层原语**——消掉 TLS / VPN / NAT 这一整摞补丁。
- **内容寻址与位置寻址并存,移动性原生**:连接绑定在身份上而非 IP,换网不断连。
- **传输与拥塞控制和第二层的织物协同设计**,从片内到广域是一条连续谱。
- 数据平面**可编程但默认内存安全**(eBPF 的能力,但是设计进去的)。

## 五、语言、编译器、运行时:一个模型从硅跑到应用

- **一个可移植、可验证的中间表示(IR)**(WASM 的思想,但覆盖从内核到应用的全谱),**把能力和那套强内存模型直接编进 IR**。一次编译,跨任意 ISA 档位运行。
- **默认内存安全的系统语言**(Rust 级的所有权,但更少附带复杂度、更符合人体工学)。性能导向:零成本抽象,系统层不强加 GC;托管运行时作为可选上层。
- **IR 携带能力与副作用(effect)信息**,于是 OS/硬件能在**零运行时成本**下强制最小权限——语言、OS、硬件**共用同一个能力模型**,端到端只有一套心智模型。
- **可复现构建是工具链的固有属性。**

## 六、数据与存储:存储是内存织物的一个层,不是另一个世界

- **默认内容寻址 + 不可变 + 版本化;默认加密、密钥归用户。**
- **存储是内存织物里的持久层**,不是「块设备 + 文件系统」叠罗汉。统一数据模型:**带历史的、类型化的、内容寻址对象**;「文件」和「数据库」退化成这层之上的两种视图。
- **结构化与可查询性是一等能力**,结构化数据不再是某个 App 的私有竖井。

## 七、身份、信任、数据归属(横切)

- **去中心化身份**:用户持密钥,身份不绑平台。
- **个人数据仓**:App 拿到的是**对用户数据的、有作用域的、可撤销的能力**,而不是各囤一份拷贝。
- 这同时是**性能与架构的简化**:更少的数据复制、更少的同步机器。

## 八、应用与交互层

- **App 天生就是能力受限、沙箱化的**——因为全栈都支持,所以这件事很便宜。
- **以共享数据模型为底的、可组合的数据中心化应用**:App 是用户自有数据之上的「视图/透镜」,而不是竖井——折叠掉今天的集成/API/同步复杂度。
- **统一的、由能力中介的扩展与自动化模型**:让脚本、自动化、AI agent 成为**一等且安全**的公民。

## 九、必须诚实面对的失败风险

- **最深的矛盾是:协同设计需要强力的前期集中协调,而集中正是扼杀采用的东西。**「worse is better / 网络效应」这一关,才是真正可能让整套东西胎死腹中的地方。
- **能力硬件有真实的硅成本**(指针变宽、标签位、面积),靠「省掉软件检查」来抵,净收益取决于工作负载。
- **一个内存模型 / 一张织物 = 更少的分叉创新空间**;靠「规范版本化 + 档位」来留演进余地。
- **处处声明式/可复现可能显得僵硬**,需要留逃生舱口。
- **处处一个 IR 有抽象税**,得靠激进的 AOT + profile 引导编译来压。
- **去中心化身份 + 个人数据仓,会撞上过去屡屡让类似尝试翻车的「密钥管理 / 用户体验」难题。**

## 一句话收束

整套体系的主线是:**围绕「能效式性能」和「消除附带复杂度」,把层与层之间的契约一起设计;让一个连贯的模型——能力 + 强内存模型 + 内容寻址数据 + 密码学身份——从硅片不间断地贯穿到应用;凡是这个模型要牺牲性能的地方,只通过明确的决策去付那个代价。** 安全、简洁、可演进,都是这条主线的副产品,而不是和性能并列的独立目标。
