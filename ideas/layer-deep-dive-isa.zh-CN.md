# 单层详解:指令集与芯片架构

[English](layer-deep-dive-isa.md)` | 中文`

> 收录于 2026 年 6 月。讨论性质,前提是先忽略硬件设备和生产制造的问题。
>
> 这是 [完整规划蓝图](clean-slate-computing-full-blueprint.zh-CN.md) 第一层的深入展开,讲到「为什么这样选、代价是什么」,而不只是列特性。

## 一、先定这一层的契约目标

ISA 本质是**软硬件之间的长期契约**。它最难改,所以要为「几十年后的高端实现」而定,而不是为最简单的实现。具体目标:

- 默认实现形态是**宽发射乱序 + SMT + 异构加速器**,不是单发射顺序核;
- 编码、寄存器、内存模型、异常模型,**全部和微架构一起定**;
- 规范是**可执行的形式化黄金模型**(类似 Sail 写的),是唯一真理来源,一致性测试和形式化验证都从它生成。

「协同设计」要落到实处,就是下面每一个决定都同时回答「软件看到什么」和「硬件好不好做」。

## 二、指令编码:为宽解码而设计

这是对 RISC-V 最直接的分歧点之一(它的 C 压缩扩展让解码变复杂)。选择:

- **主指令固定 32 位**,让「切指令边界」是平凡的——N 路解码器可以无依赖地并行切分,这是宽前端的命门。不要 x86 那种 1~15 字节、必须串行扫描才能定边界的变长。
- **要密度,但不靠任意变长。** 用一个**解码友好的「打包(bundle)」机制**:长度信息固定编码在每个槽的固定比特位,解码器不解析内容就能知道长度。
- **寄存器字段位置在所有指令格式里固定**(这一点 RISC-V 做得好,直接借):寄存器读可以在完整译码完成前就启动。
- 给**架构级的融合指令**留正规编码,让「融合」是软件可见的契约,而不是各家微架构私下做、彼此行为不一致的 macro-op fusion。

代价:固定 32 位比极致变长的代码密度差一些(指令 cache 压力略大),但换来确定性的高带宽解码。对高端目标这笔账划算。

## 三、寄存器与「要不要标志位」

- **32 个通用寄存器**起步。再多能减少 spill,但吃编码位、也加大上下文切换状态;32 是高端 OoO 上经过验证的甜点。物理寄存器靠重命名堆。
- **不设全局标志寄存器(flags)。** 全局 flags 在重命名时会制造「部分标志位」假相关,是乱序设计里出了名的麻烦(x86 的历史包袱之一)。
- **但补回它的好处**:提供**架构级的 compare-and-branch、compare-and-select(条件选择)**作为单条指令。既没有全局 flags 的重命名污染,又不需要靠微架构去猜测性融合——融合写进了 ISA。

## 四、内存一致性:一个模型,且「默认就对」

这是「让最快的路就是最对的路」原则最关键的落点。

- **只有一个模型,形式化规范死**,不给「弱/强可选」。
- **模型是多副本原子(multi-copy atomic)的 release/acquire 体系(RCsc 风格)**:普通 load/store 带 acquire/release 语义即可表达绝大多数同步,语义清晰、可被工具验证。
- **默认强、opt-in 弱**:不带标注的访存走较强语义,**naive 代码天然正确**;真正要榨性能的专家可以显式用 relaxed 变体。

和两个替代摆明取舍:**TSO(x86 那种)**更强、更好编程,但给微架构的乱序自由更少、跨核一致性流量更大;**ARM/RVWMO 那种弱模型**自由度大但太难推理,容易写出隐蔽 bug。选 RCsc + 默认强,是偏向可推理性的折中——和性能优先并不冲突,因为难推理的 bug 本身就是巨大的隐性成本。

## 五、内存安全:纯能力架构(CHERI 路线)

这是和上层 OS/语言能力模型**同构**的地基。

- **所有指针都是能力(capability),纯能力架构**,不搞「整数指针 + 能力」的混合(混合是为了兼容老代码,我们没有这个包袱)。一个能力 = 地址 + 边界 + 权限,**128 位**,外加**每个能力槽 1 位标签**存在旁路(tag,防伪造)。边界用压缩编码(CHERI Concentrate 那类)塞进 128 位。
- **回报是性能 + 正确性双赢**:空间安全(越界)在硬件上不可能发生,**省掉软件里铺天盖地的边界检查指令**;同时它直接当作进程间/模块间隔离的硬件基元。
- **诚实讲清两笔代价**:
  1. **指针变宽到 128 位** → cache 和内存占用上升、指针密集结构带宽压力大。这是这条路线最实在的成本。
  2. **时间安全(use-after-free)不是免费的**:空间安全靠能力自然解决,但悬垂引用要额外机制——能力撤销 + 周期性清扫(CHERIvoke 那类)或加载屏障,有运行时开销。
- **协同设计的红利**:OS 的能力、语言 IR 里的能力,**一路下沉到这同一个硬件能力**,不是软件模拟——端到端只有一套保护模型。

## 六、保护模型:用「持有能力」取代「特权环」

- 抛弃 x86 的 ring0~3 / 复杂特权层级。**保护主要靠「你是否持有相应能力」,而非「你在哪个特权级」**,模型更扁平、更可组合。
- 仍保留**极少量的非对称**(给那个形式化验证的微内核一个最小可信根),其余全部能力化。
- **精确异常 + 结构化异步**:乱序实现也必须给出精确异常(可调试、可恢复);中断走和 OS 那层「批量异步 ABI」协同设计的结构化模型。

## 七、数据并行:标量/向量/张量统一,不是三座孤岛

- **向量长度无关(VLA)**(SVE / RVV 的正确思路):代码不写死向量宽度,同一份二进制在不同实现上自动伸缩。
- **掩码/谓词是一等公民**,不是后补。
- **张量/矩阵作为一等扩展**,和标量、向量共享同一套寄存器与内存模型。这里有个真实难题:矩阵单元状态巨大,**怎么虚拟化和上下文切换**?取向是**让矩阵指令作用在「内存描述符」上而非一块巨大的架构寄存器堆**,把可切换状态压到最小——拿可切换性换一点峰值便利。
- 三者统一在一个模型下,而不是像今天 CPU 标量、GPU、NPU 各说各话。

## 八、异构:一个地址空间,加速器是平权公民

- **CPU/GPU/NPU/DPU 全是相干织物上的 agent,共享同一个虚拟地址空间,连能力一起共享。** 页表/地址翻译统一,IOMMU/MMU 合一。
- 后果:**取消「设备内存」这个独立 API**,取消跨 PCIe 拷贝;能力可以**安全地跨越加速器边界**传递——这是上层「统一内存模型」承诺能成立的硬件前提。

## 九、用「档位 Profile」消灭碎片化

- 不走「最小基础集 + 一堆可选扩展」。定义**少数几个完整档位**(例如:嵌入式级 / 应用级 / 服务器·HPC 级),**每个档位是保证齐全的一整套**,带版本号。
- 「这颗芯片是应用级 v2」就等于一组确定、完整、可依赖的能力,软件不必面对「理论上有、这颗没有」的 à la carte 噩梦。

## 十、规范即真理:形式化黄金模型

- 用可执行形式化语言(Sail 类)写**唯一权威规范**;一致性测试**从规范自动生成**;硬件实现和那个验证内核**对着同一个模型做形式化验证**。
- 这让「一致性」从「跑一堆测试碰运气」变成「对规范可证明」。

## 十一、这一层特有的、必须认账的取舍

- **纯能力 128 位指针**:内存/cache 压力是真实且持续的成本,指针密集型负载最吃亏。
- **时间安全**还没有零成本方案,撤销/清扫有开销。
- **矩阵单元的虚拟化**:用内存描述符压状态,牺牲了一点峰值编程便利。
- **单一内存模型**:可推理性换来的,是放弃了一部分微架构可能想要的激进重排自由。
- **固定 32 位编码**:解码简单的代价是密度不如极致变长。
- **统一异构地址空间**:相干性要覆盖到加速器,**相干织物的设计复杂度和扩展性压力**被推到了第二层(内存与互连),那层会更难做。

## 一句话收束

把「宽乱序 + 异构 + 能力安全」当成 day-one 的设计目标,让编码、寄存器、内存模型、保护模型彼此协同;凡是要牺牲性能或便利的地方(128 位指针、单一内存模型、固定编码),都只在明确认账后才付。
