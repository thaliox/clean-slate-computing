# 单层详解:内存与互连

[English](layer-deep-dive-memory-interconnect.md)` | 中文`

> 收录于 2026 年 6 月。讨论性质,前提是先忽略硬件设备和生产制造的问题。
>
> 这是 [完整规划蓝图](clean-slate-computing-full-blueprint.zh-CN.md) 第二层的深入展开,直接承接 [ISA 层详解](layer-deep-dive-isa.zh-CN.md) 甩过来的难题:统一异构地址空间要求相干性覆盖到加速器,这一层负责兑现。

## 一、这一层必须兑现的契约

上一层(ISA)做了三个承诺,账都记在这一层头上:

- **一个相干、字节寻址的地址空间**,横跨所有计算单元(CPU/GPU/NPU/DPU)和所有内存层级(高带宽层 / 主存层 / 持久层);
- **能力(capability)和标签(tag)要能在内存里活下来、传下去**;
- **和网络层是一条连续谱**——从片内 NoC 到机架到数据中心,语义统一。

最硬的一块:**相干性要扩展到成百上千个 agent + 大带宽加速器**,而经典相干根本扩不到这个规模。这一层成败就在这里。

## 二、核心难题:相干性扩不上去

全硬件 cache 相干(snoop / 目录)是最好编程的模型,但**目录状态和一致性流量随 agent 数量爆炸**,扩不到上千节点 + 加速器。今天 GPU 之所以用松散/带作用域的相干,正是这个原因。

我的选择是**分层 + 带作用域的相干**,而且关键是**和 ISA 的内存模型协同**,不是另起炉灶:

- **相干域(coherence domain)内**:全硬件相干、快——一个节点/插槽/少数几插槽是一个域,naive 代码在域内天然正确。
- **跨域**:需要在正确的**作用域(scope)**上显式 acquire/release。这个"作用域"是**架构层面对软件可见的概念**,不是藏在微架构里的黑魔法。
- 目录用**稀疏目录 + 层级化**,相干消息走下面那张织物。
- 总思路还是"**域内强、跨域显式**",和 ISA 那条"默认强、opt-in 弱"完全一致。

## 三、内存模型的延伸:作用域(scope)

ISA 那篇的"一个内存模型"怎么在异构织物上活下来?靠**给 release/acquire 加作用域**:

- 定义层级作用域:**线程 / 核 / 相干域 / 全系统**。
- 一次 release/acquire 标注它作用到哪一层(类似 GPU 的 scoped memory model,但**统一、且形式化规范死**)。
- 这样既不要求"全局相干"这种扩不动的东西,又让程序员只面对一套带作用域的 release/acquire 规则——可推理性保住了。

这是"一个内存模型"在物理上能成立的唯一办法:**把"距离"变成作用域参数,而不是变成另一套语义**。

## 四、地址翻译与标签存储

- **一个虚拟地址空间,翻译统一**:CPU 和所有加速器共享页表/翻译,IOMMU 与 MMU 合一。能力工作在虚拟空间,翻译在其下。
- **规模化翻译的真实成本**:加速器也要做地址翻译(类似今天的 ATS/PRI),**TLB 压力**大;靠大页、范围翻译、翻译缓存共享来压。
- **标签存储是这条路绕不开的账**:每个 128 位能力槽 1 位 tag,约 **0.8% 的内存开销**。实现上要么塞进 DRAM 的 ECC 旁路,要么单开一块**标签表 + 标签缓存(tag cache)**。tag 必须在物理内存里也活着、且不可被普通写伪造——这是纯能力架构能成立的硬件前提。

## 五、内存层级:一个地址空间 + 放置提示

- **高带宽层、主存层、持久层都在同一地址空间里**,不是各搞一套 API。
- **放置靠提示/策略**,层间迁移由硬件 + OS 处理(类似 autonuma,但是设计进去的,不是事后补丁)。
- **持久性做干净**:定义**持久化域**,把 flush/commit 的次序语义**并进内存模型**(acquire/release + 一条 persist/commit 排序)。今天 pmem 编程模型之所以痛苦,就是持久次序是事后硬塞的;这次它从一开始就和一致性模型长在一起。

## 六、近存计算:省数据搬运就是省能耗

这条直接服务总纲里的"能效优先"。

- **搬数据比算数据贵得多**(一次跨芯片搬运的能耗远超一次算术运算),所以织物要允许**把计算推到数据旁边**。
- 暴露**近存原语**:内存端原子操作、归约、scatter/gather,乃至小规模 offload。
- 这是最大的能耗杠杆之一:很多负载的瓶颈和耗电都在数据移动,不在算力。

代价:近存计算会**让一致性故事变复杂**——在内存端发生的计算也必须参与内存模型,不能是模型外的"幽灵写"。

## 七、互连:一张分层织物取代动物园

把今天 PCIe / 以太 / NVLink / CXL / InfiniBand 这一堆**合并成一张分层织物**:

- **分层**:物理/链路层(SerDes/lane,这里忽略制造)+ **事务层**。事务层统一承载:相干内存事务、单边 RDMA 式操作、消息传递——**全部带身份和能力**。
- **交换式,不是总线**;拓扑灵活(mesh / dragonfly 等)。
- **自带身份**:每个 agent、每个包都可认证(接上网络层的身份原语)。**DMA 受能力检查**——加速器只能碰它持有能力的内存,从根上消灭今天的 DMA 攻击/越权那一整类。
- **连续谱**:从片内 NoC 到机架到数据中心,**事务模型同一套**;延迟和带宽不同,但**语义一致**。"距离"是性能属性,不是语义属性。

## 八、拥塞、QoS、隔离

- 织物规模上必须有**和传输层协同设计的拥塞控制**:基于信用的流控、网内拥塞信号。
- **租户间 QoS 与隔离**:能力 + 身份天然支撑——谁能用多少带宽、能不能碰某段内存,都是能力说了算。

## 九、规模化可靠性

- 组件越多,故障越多。**端到端完整性**(校验、ECC)是底线;上层(第六层)内容寻址 + 不可变数据模型能帮上忙(数据可校验、可重取)。
- 相干织物必须**优雅处理链路故障/重传**,有清晰的**故障域**划分,一个域挂掉不连累全系统。

## 十、必须认账的取舍

- **核心张力:全相干扩不动 → 带作用域的相干把复杂度推回给软件。** 虽然靠"和内存模型协同"缓解,但程序员终究要面对作用域,这是真实成本。
- **统一地址空间 + 能力 + 标签 = 翻译和标签存储开销**,TLB/标签缓存压力实打实。
- **一张织物是巨大的协调/标准化赌注**——"worse is better / 既得利益"的风险在这一层尤其尖锐,因为现实里互连标准林立、各有山头。
- **近存计算让一致性更复杂**:内存端的计算得乖乖进模型。
- **持久次序**给本来干净的内存模型又加了一层复杂度。
- **延迟连续谱"语义统一但性能不统一"**:程序员可以写出"正确但慢到灾难"的、无视局部性的代码(行星尺度的 NUMA 问题)。必须配好的**局部性提示和可观测性**,否则正确性有了、性能塌了。

## 一句话收束

这一层的活,是**把"一个相干地址空间"这个美好承诺,在扩不动的物理现实里兑现**:用"域内强相干 + 跨域带作用域"接住 ISA 的内存模型,用"一张带身份和能力的分层织物"取代互连动物园,用近存计算省下作为能耗大头的数据搬运;凡是物理逼着要牺牲的地方(作用域复杂度、标签开销、性能非均匀),都摆到明处、由软件显式面对。
